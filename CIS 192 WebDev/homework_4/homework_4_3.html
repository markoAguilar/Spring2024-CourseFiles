Part 3: Common Form Widgets

A checkbox input element initially appears selected if the checked attribute is set. The checked attribute is an example of a boolean attribute which is true if it is present and false if it is absent! No value must be assigned to a boolean attribute!

For each checkbox selected, the browser sends the checkbox's name and value to the server. value attribute default is "on". If a checkbox is not selected, the browser does not send anything to the server which is a common error for servers since it fails to record all checkboxes.

Good practice is to use label elements with checkboxes so the user can click the label to check and uncheck the associated checkbox. Try it out below!

Radio Button

A radio button is a widget for input elements with the type attribute "radio", which allows users to select exactly one value from possibly many values. The web browser groups radio buttons together with the same name attribute, where each possible value in a group has an associated input. When submitting a form. the browser sends the selected radio button's name and value attribute. Ex: If the radio button <input type="radio" name="movie" value="ET"> is selected, "movie=ET" is sent to the server.

The main difference between radio and checkbox is that only one radio button in a group can be selected, while any number of checkboxes can be selected. Furthermore, the browser requires one radio button always be slected form that group. Therefore, the browser will minimally send the selected radio button if any. Try It Below!.

Drop-Down Menu

The <select> container creates a drop-down menu or list, which allows user to select one of several predefined values. The <option> container creates a value, or option, the user can select within a drop-down menu. When the user is not interacting with the menu, the drop-down menu usually displays the selected option.

The difference between a drop-down menu and a radio button is that the drop-down menu only displays the options when interacting with the user, while a radio button always displays all options.

List Box

A list box widget is created by specifying a size with the select element's size attribute. Ex: <select size="4"> creates a list box that shows four options at a time. If the list box contains more than size options, the browser adds a vertical scrollbar so the user can scroll through the list of options.

The multiple attribute allows the user to select multiple options. On windows, the user must hold down the control button to select multiple options, and on a Mac, the user must hold down the command button. Many users are unaware of this so best practice is to use checkboxes in that case.

  • a password field is a widget element for the <input> tag
  • set with <input> type attribute value of "password"
  • forms that submit sensitive data should always submit with URL's that use HTTPS
  • size and maxlength attributes are used to set the maximum display size and the maximum amount of characters allowed for an entered password, respectively.
