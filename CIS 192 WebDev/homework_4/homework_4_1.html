Containers And Parent Containers

A container is any html tag in the body of an html document. Developers use these to assist in
organizing and formatting content.

In the table below, the <ol> container would be the parent of the list elements.
However it would not be the parent to the <img> elements.That would be the <li> container!
Further note that the image tag is not a container since it does not have a closing tag!

Block Elements

A block element fills the width of the elements parent container and can container other block elements, inline elements, and text.
Block elements include <h1>, <table>, and <p>. The p element cannot contain another p element because the semantics are unclear.

The <div> container tag is the only block element with no semantic meaning. It is a generic element for creating block containers for presentational and functional purposes.The following HTML breaks up a document into logical components use a <div> tag for each component.

<body>
<div>
Logo and headline elements
</div>
<div>
Navigational elements
</div>
<div>
Main content
</div>
<div>
Footer and legal
</div>
</body>

Inline Elements

An inline element fills the minimum space possible in the elements's parent container and can only contain text or other inline elements. Inline elements do not have closing tags. An example of an inline element is the <a> tag. It creates a hyperlink container to hold the link and the hyperlink does not fill the width or height of the link's parent paragraph.

The <span> element is a generic inline element. Unlike other inline elements, such as <a> and <em>.

Since div and span do not have semantic meaning, they are used primarily for presentation and interaction purposes. Good practice is to use these tags as a last resort. Attempt to use tags such as <address> or <article> where appropriate and possible.
