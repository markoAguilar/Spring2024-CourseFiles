Audio Element

The <audio> container plays an audio file within a webpage. It contains within it a <source> element with the value of its src attribute set to the location of the audio source to play.

Common <audio> attributes include:

  • autoplay: boolean, makes the audio play automatically
  • controls: boolean, displays audio controls for user to pause, play, and control volume.If not present, user cannot pause or control audio!
  • loop: boolean, replays audio at end of audio
  • muted: boolean, initially mutes the audio
  • Note: since the above are all boolean attributes they are either true when present and false when absent

Important notes: some browsers may not allow audio to play automatically regarnding user settings since some users do not prefer autoplay! Furthermore, developers used to use <embed> and <object> elements to embed audio in a webpage. These would require a plug-in however to play such as Flashplayer to play flash files. But, plug-ins are a source of security risks and some browser have thus stopped supporting some plugins.

Common Audio Formats

Different web browsers support different audio formats. Multiple <source> elements can be used to support alternate formats.

  • AAC: Advanced audio coding, (.aac) designed to be the succesor to mp3 with better qualityi popular
  • MP3: (.mp3), the sound portion of a MPEG file. widest support, most popular
  • Ogg Vorbis: (.ogg), open-source audio coding format developed by Xiph.org
  • Wave: (.wav), developed by IBM and Microsoft, mainly for storing uncompressed audio on windows

Video Element

Just like the <audio> container, the <video> container contains a <source> element whose src attribute is assigned the location of the video file to play

Some common <video> attributes include:

  • All the boolean attributes of an <audio> container
  • width: specifies the pixel width of the video's display

Note that some browsers will not play a video automatically unless it is muted!

Common Video Formats

Different web browsers support different video formats. Multiple <source> elements can be used to support alternate formats.

  • MPEG-4 or MP4: (.m4A, .mp4), m4A is audio only
  • Ogg Theora: (.ogg, .ogv, .ogm), open video compression format developed by Xiph.org foundation
  • WebM: &40;.webm), open media format developed by Mozilla, Opera, Adobe. and Google for the web

Showing YouTube Videos!

Watch this funny video!

The <iframe> container allows a webpage to be embedded in a rectangular area of the current webpage. It is through this embedding that a YouTube video may be played!
The <iframe> container has attributes:

  • src: assigned the location of the video to be played. Note that this means the <iframe> container does not need a <source> element
  • height and width: self explanatory
  • allow: assign the value "autoplay" to autoplay the video
    • Note however that sometimes this may not work and you would thus need to append &autoplay=1 to the end of the arguments in the URL

Visit the YouTube help pages for learning how to upload and embedd a YouTube video!
