HTML Developer Guidlines

Apparently it is bad practice to use self-closing tags! For example <br />. So I need to configure the prettier linter that I use for VIM since it applies that change automatically!

Another good practice is to always use quotes for attribute values. And when using quotes to always use double quotes, which I have been doing thus far&46; Furthermore, if you need to use double quotes within a quoted phrase you need to escape the double quote, or any special character, with a backslash \

  • Start block elements on a new line. Place inline elements in newlines only for code readability purposes
  • Indent nested elements
  • Do not use embedded or inline CSS or Javascript use the class attribute and assign it the CSS class that you have implemented in your externall CSS file!
    • To do this simply use <link rel="stylesheet" href="cssFileName.css">
