Part 1: Using CSS In HTML

Introduction To CSS

CSS (Cascading Style Sheets) describes how the webpage is styled for visual presentation. The applied style rules cascade downfrom general to more specific with the more specific rules overriding more general rules

A CSS Rule consists of:

  • A selector: Specifies the HTML elements to which the style rules apply
  • A declaration block: Contains one or more declarations separated by semicolons (;)
  • Styling Declarations: Each styling declaration is a line with a key:value pair and ending with a semicolon (;)

Applying CSS

CSS can be applied by:

  • Inline Declarations: Inline declarations are written as the value to the style attribute and are also key: value; pairs inside quotes. No curly braces required.
  • Embedded CSS: Embedded css is written between two <style> tags in between the <head> tags of your html document
  • External Style Sheets: This is a separate files that is linked to your html document within the head tags using the format: <link rel="stylesheet" href="cssFileName.css">

Inheritance And Conflict

When two style declarations with identical properties apply to the same element a conflict occurs. Two common types of conflicts are:

  • A parent's declaration conflicts with a child's declaration: When this conflict occurs, the child's declaration overrides the parent's declaration
  • An embedded or external style sheet conflicts with an inline declaration: When this conflict occurs, the inline declaration overrides the external or embedded declarations

Specificity and !important

A browser uses specificity which is like a scoring system to apply css rules appropriately. In more complex cases you may wish to override this. To do so you would need to add the !important rule right before ending a declaration line. This overrides all other declarations. However, this is not good practice as implementation changes can easily become overly complex!
